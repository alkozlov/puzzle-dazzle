# Основные изменения

Цель данного изменения - добавить возможномть проходить созданный лабиринт. Это позволит пользователям не только создавать лабиринты, но и испытывать их на практике, что повысит вовлеченность и удовлетворение от использования приложения.

## Изменения в функциональности

1. **Добавление режима прохождения лабиринта**: В интерфейсе приложения будет добавлена новая опция, позволяющая пользователям переключаться в режим прохождения лабиринта. В этом режиме пользователи смогут управлять точкой входа в лабиринт и пытаться найти выход, используя клавиши управления или сенсорные жесты. Данная опция доступна после создания лабиринта и будет отображаться в правом нижнем углу UI элемента, который отображает лабиринт. Кнопка будет содержать иконку двери, символизирующую вход в лабиринт.

2. **Экран прохождения лабиринта**: При нажатии на кнопку входа в лабиринт, пользователи будут перенаправлены на новый экран, который отображает лабиринт в более крупном масштабе. При этом этот экран будет переведен в landscape режим. Лабиринт должен занимать большую часть экрана, чтобы обеспечить удобство навигации. В правом нижнем углу будет отображаться своего рода джойстик для управления движением по лабиринту. Пользователи смогут перемещаться в четырех направлениях (вверх, вниз, влево, вправо) с помощью этого джойстика.

3. **Перемещение по лабиринту**: Пользователи смогут перемещаться по лабиринту, используя джойстик. Перемещение начинается в точке входа, которая представлена зеленой точкой. Целью является пройти до красной точки, которая символизирует выход из лабиринта. При достижении красной точки пользователи будут получать уведомление о том, что они успешно прошли лабиринт и увидят время, затраченное на прохождение. Это может стимулировать пользователей создавать более сложные лабиринты и соревноваться с друзьями.

4. **Принудительное завершение режима прохождения**: Если пользователь хочет выйти из режима прохождения лабиринта, он может нажать на кнопку "Выход" в правом верхнем углу экрана. Это позволит пользователю вернуться к режиму создания лабиринта и продолжить работу над ним или создать новый лабиринт. Если пользователь покидает режим прохождения, все его достижения в текущем прохождении будут сброшены, и он начнет новый сеанс прохождения при следующем входе в режим прохождения.

## Особенности управления

**Джойстик**: Джойстик для управления движением по лабиринту будет интуитивно понятным и отзывчивым, обеспечивая плавное перемещение. Он будет отображаться в правом нижнем углу экрана и будет достаточно большим, чтобы пользователи могли легко управлять им, особенно на мобильных устройствах. Пользователю будет доступно 4 направления для перемещения: вверх, вниз, влево и вправо. Джойстик будет работать на основе касаний или кликов, в зависимости от устройства, и будет обеспечивать мгновенный отклик на действия пользователя. При нажатии на соответствующую область джойстика, зеленая точка будет перемещаться в выбранном направлении, если это возможно (т.е. если нет стены на пути). Если пользователь пытается переместиться в направлении, где есть стена, джойстик не будет реагировать, и зеленая точка останется на месте.

**Трасировка пути**: Во время прохождения лабиринта, пользователи смогут видеть свой текущий путь, который будет отображаться в виде линии, соединяющей все точки, через которые они прошли. Это поможет пользователям ориентироваться в лабиринте и планировать свои движения. Линия пути будет отображаться на фоне лабиринта, однако она не должна иметь слишком яркий цвет, чтобы не отвлекать от основного дизайна лабиринта. Цвет линии пути может быть, например, светло-серым или голубым, чтобы обеспечить хорошую видимость без излишней яркости. Если пользователь возвращается по своему пути, линия будет обновляться, удаляя участки, которые были пройдены назад. Это позволит пользователям видеть только актуальный путь, который они проходят в данный момент.
